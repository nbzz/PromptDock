```yaml

    type: stock
    required: true
    placeholder: 输入股票代码或名称，如：贵州茅台
  预测年数:
    type: select
    options: ["3", "5", "7"]
    default: "5"
  收入增长率:
    type: text
    default: "15%"
    placeholder: 如 15%
  EBITDA利润率:
    type: text
    default: "20%"
    placeholder: 如 20%
  资本支出强度:
    type: text
    default: "5%"
    placeholder: 如 5%（Capex/Revenue）
  营运资本变动:
    type: text
    default: "2%"
    placeholder: 如 2%（NWC/Revenue）
```

**1. 利润表预测**
- 收入预测：收入=上期收入×(1+增速)
- 营业成本：成本率假设（历史均值或给定值）
- 毛利率/EBITDA率逐年列示
- 折旧摊销假设（D&A/Revenue历史均值）
- 税率假设
- 输出完整年度利润表表格

**2. 资产负债表预测**
- 资产侧：应收/存货/固定资产（原值-累计折旧+新增Capex）
- 负债侧：应付/短期借款/长期借款
- 权益侧：期初权益+净利润-分红
- 平衡检验：资产总计=负债+权益

**3. 现金流量表预测**
- 经营现金流：净利润+折旧摊销-营运资本增加

**4. 三表勾稽关系验证**
- 净利润→经营现金流差额（应计项目）
- 固定资产原值变化=期初+新增-折旧
- 现金变动=三部分现金流之和

**5. 关键比率分析**
- ROE=净利润/平均股东权益
- 资产负债率=总负债/总资产
- 现金流覆盖率=经营现金流/流动负债

## 跨文件共性问题

### 1. `[变量]` 占位符与 Markdown 语法冲突
**影响范围：** Competitive_、Comps_、DCF_、LBO_、个股分析、企业竞争优势分析 等多个文件

多个文件在正文中使用 `[变量名]` 作为变量占位符（如 `[目标公司]`、`[竞争对手列表]`），这与 Markdown 的一级标题语法 `#` 冲突——当用户渲染这些 .md 文件时，所有以 `#` 开头的行会被解析为标题，而 `[变量]` 会被直接显示为普通文本而非动态变量。

**建议统一修改为：** 使用 `{{变量名}}` 或 `VAR(变量名)` 等不易与 Markdown 语法冲突的格式。

### 2. 变量定义不一致
- 部分提示词有完整的 `variables:` YAML 块
- 部分提示词（如 ThreeStatement、学术论文结构化阅读）直接在正文中使用 `[变量]`
- 部分提示词的 YAML 块缺少 `placeholder` 字段（体育营销论文、西甲体育营销方案等）

**建议：** 统一要求所有提示词必须包含完整的 `variables:` YAML 块，且每个变量必须有 `type`、`required`（可选）、`placeholder` 三个基础字段。

### 3. 提示词之间功能重叠
- `News_财经新闻分析.md` 与 `由新闻分析个股板块影响.md` 功能高度重叠，后者更完整
- `西甲体育营销方案.md` 与 `通用型 品牌体育营销.md` 约90%内容重叠
- `ReportBuilder_分析报告生成.md` 与 `个股分析.md` 存在一定重叠

**建议：** 考虑合并功能重叠的提示词，明确各自的定位差异。

### 4. 变量 autoFill 的实现依赖
多个提示词使用了 `autoFill: date`，但当前 PromptDock 的 autoFill 功能仅支持日期类型。如果未来扩展到其他类型（如 `autoFill: stockInfo` 获取实时股价），现有提示词中没有对应的框架支持。
