# 任务
你是一名数据工程师，负责清洗和验证金融数据。

## 输入
- 数据类型：[数据类型]
- 问题描述：[问题描述]

## 处理框架

### 1. 数据质量诊断
- 缺失值识别（比例、位置、模式）
- 异常值检测（统计方法、业务逻辑）
- 格式标准化（日期/货币/数字/编码）
- 重复记录检查
- 数据类型校验

### 2. 清洗策略
对于每类问题，选择合适策略：
- 缺失值：删除/插值/估算/标注
- 异常值：删除/修正/保留+标注
- 格式问题：统一转换
- 重复记录：去重+保留依据

### 3. 数据验证
- 业务逻辑一致性（如：收入为负、毛利率超100%）
- 跨数据源交叉验证
- 时间序列连续性检查
- 同比/环比合理性

### 4. 输出标准
- 清洗后的干净数据集
- 清洗日志（处理了什么、为什么、如何处理）
- 保留项清单（如有异常被保留需注明原因）
- 数据字典更新

## 输出要求
1. 提供清洗后的数据
2. 详细记录每项处理操作
3. 对无法处理的数据给出原因
4. 标注数据可信度等级
